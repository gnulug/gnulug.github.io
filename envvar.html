Environment Variables Reference Guide

How to read environment variables

C:

#include <stdio.h>
#include <stdlib.h>

int main() {
	char *path = getenv("PATH");
	printf("%s\n", path);
	return 0;
}

Bash:

echo $PATH

Python:

import os
print(os.environ['PATH'])

Find current environment variables

  • Global: using the env or printenv commands to list all exported environment variables
  • For a specific process: cat /proc/$$/environ

A list of environment variables

Starts with commonly used ones and gets more obscure as we go down the list.

  • PATH: a colon separated list of directories to look for executable binaries.
  • PWD: current working directory
  • HOME: home directory
  • USER: current user name
  • SHELL: default shell
  • TERM: terminal type
  • EDITOR: default editor
  • LANG: default language
  • CDPATH: a colon separarated list of directories for cd to look at when changing directories. If unset, it is usually just . (i.e. current working directory.
  • MANPATH: a colon separated list of directories to look for man pages.
  • OLDPWD: the previous working directory. Used by cd -.
  • SHLVL: the process level of the current shell.
  • PAGER: for example, can change to more.
  • POSIXLY_CORRECT: Enforces POSIX correctness for the output of some utilities, for example, df.
  • COLUMNS and LINES: number of columns and lines of characters available in the current terminal.
  • LD_LIBRARY_PATH: a colon separated list of directories to look for libraries to load.
  • LD_PRELOAD: a library to load before running a program.
  • MACHTYPE: current system architecture (i386, x86_64 etc).

Bash related tricks:

Credit to GNU Bash Manual and StackOverflow

  • $0, $1, ...: arguments passed to the current running script/shell
  • $PS1: describes the prompt.
  • $$: current shell's PID.
  • $_: last argument passed to the last running command.
  • $?: exit code of last running command.
  • $!: PID of process most recently sent to background.
  • $#: number of arguments passed to shell script.
  • $@: enumerates all arguments passed to shell script.
  • $IFS: specifies the delimiter for commands such as read.
